# IAM (Identity and Access Management) 

Gerenciar identidades e permissões de acesso aos recursos da nuvem, ou seja, controla quem pode fazer o quê dentro da sua conta. 

<u>Importante:</u>

- Criação da conta secundaria para uso diário, aplicar o MFA 
- IAM é global, ou seja, não depende de região. 

 
## Usuários (Users) 

- Criação de usuários, cada um com suas próprias credenciais.  
- Credenciais próprias (senha ou chaves de acesso para usar os serviços). 
- Para facilitar o gerenciamento de permissões, organizar os usuários em grupos. 

## Grupos (Grups) 

- Conjunto de usuários (exemplo: “ADM”, “ Devs”). 
- Simplifica o gerenciamento de políticas, pois os usuários do grupo herdam as permissões. 

## Políticas (Policies) 

- Regras que dizem o que pode ou não pode ser feito cada usuário ou grupo. 
- Exemplo: “Pode criar instâncias EC2” ou “Pode apenas ler objetos no S3”. 

## Funções (Roles) 

- Identidades temporárias com permissões específicas entre os recursos da AWS. 
- Deve ser usada o mínimo possível (só para tarefas críticas, como mudar informações de cobrança).
