# Configuração da conta 


## Conta Raiz (Conta Root)  
- Conta com super privilégios, ou seja, acesso total dos recursos e configurações. 
- Recomendado criar um usuário administrativo separado no IAM com alguns privilégios para uso diário. 


# Práticas de Segurança 

-Não usar a conta root no uso diário. 

- As credenciais da conta root devem ser guardadas, usar apenas quando for muito necessária. 
- Utilizar o Autentificação Multifator (MFA), principalmente na conta root. 
- Criar políticas de permissão com o princípio do menor privilégio (apenas as permissões necessárias). 
- Usar o AWS IAM (Identity and Access Management) para gerenciar usuários, grupos e políticas de acesso.
